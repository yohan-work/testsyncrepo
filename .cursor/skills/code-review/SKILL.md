# 코드 리뷰 스킬

## 리뷰 프로세스

1. **변경사항 파악**: `git diff` 또는 PR diff 확인
2. **체크리스트 적용**: 아래 항목 순차 검토
3. **피드백 작성**: 우선순위별 정리

## 리뷰 체크리스트

```
코드 리뷰 체크리스트:
- [ ] 코드가 요구사항을 충족하는가
- [ ] 로직이 정확하고 엣지케이스를 처리하는가
- [ ] 함수/변수 이름이 명확한가
- [ ] 중복 코드가 없는가
- [ ] 적절한 에러 처리가 있는가
- [ ] 보안 취약점이 없는가
- [ ] 테스트가 충분한가
- [ ] 성능 이슈가 없는가
```

## 피드백 형식

```markdown
## 리뷰 결과

### 🔴 Critical (반드시 수정)
- [파일:라인] 이슈 설명
  ```수정 제안```

### 🟡 Warning (수정 권장)
- [파일:라인] 이슈 설명

### 🟢 Suggestion (개선 제안)
- [파일:라인] 개선 아이디어
```

## 코드 품질 기준

| 항목 | 기준 |
|------|------|
| 함수 길이 | 50줄 이하 |
| 파라미터 수 | 4개 이하 |
| 중첩 깊이 | 3단계 이하 |
| 복잡도 | 10 이하 |

## 사용 예시

```
code-reviewer 에이전트를 사용해서 이 PR을 리뷰해줘
```
